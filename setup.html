🕊️ FreeME - Setup Guide

Follow these steps to set up your own FreeME social media platform on GitHub Pages.

Step 1: Create a Firebase Project

  1. Go to Firebase Console
  2. Click "Create a project" (or use an existing one)
  3. Give it a name (e.g., "freeme-social")
  4. Disable Google Analytics (optional) and click Create

Step 2: Enable Authentication

  1. In Firebase Console, go to Authentication → Sign-in method
  2. Enable Email/Password
  3. Enable Google (optional, for Google sign-in)

Step 3: Create Firestore Database

  1. Go to Firestore Database → Create database
  2. Select "Start in test mode" for now
  3. Choose a location closest to you
  4. Later, copy the security rules from firestore.rules.js into Firestore → Rules

Step 4: Enable Storage

  1. Go to Storage → Get started
  2. Accept the default rules for now
  3. This is used for uploading profile pictures and post images

Step 5: Get Your Firebase Config

  1. Go to Project Settings (gear icon) → General
  2. Scroll to "Your apps" → Click the web icon </>
  3. Register your app (name: "FreeME")
  4. Copy the firebaseConfig object
  5. Open js/firebase-config.js and replace the placeholder values

Step 6: Create Firestore Indexes

Firestore will auto-create indexes when needed. If you see index errors in the browser console, click the link provided to create them automatically.

Required composite indexes:

  • posts → authorId (ASC) + createdAt (DESC)
  • notifications → targetUid (ASC) + createdAt (DESC)
  • notifications → targetUid (ASC) + read (ASC)

Step 7: Deploy to GitHub Pages

  1. Create a new GitHub repository
  2. Push all these files to the repo
  3. Go to repo Settings → Pages
  4. Under "Source", select main branch and / (root)
  5. Click Save
  6. Your site will be live at https://yourusername.github.io/repo-name

Step 8: Add GitHub Pages URL to Firebase

  1. Go to Firebase Console → Authentication → Settings → Authorized domains
  2. Add your GitHub Pages domain: yourusername.github.io

Important Notes

  • First user to register becomes Admin automatically
  • Admins can manage users, posts, reports, and site settings from the Admin panel (shield icon)
  • Images are stored in Firebase Storage (or base64 fallback)
  • Embeds support YouTube, Vimeo, Spotify, and generic links
  • Dark mode is available in Settings
